Auto-implemented properties

Auto-implemented properties are an easier way to write getters and setters. Instead of doing GetX and SetX for a X properties defined, one can simply use public int X { get; set; } and voila. C# compiler automatically defines the private field and the (public) property.

Every (private) field should have a (public) property that modify/ read it. As mentioned here, when no further validation will take place, auto-implemented properties are more than welcome.

Finally, one can implement access modifiers on getters and setters.
