Property vs Field

There are basically two types of class-defined variables: properties and fields. Fields should be always private to the class and will contain the actual logic and data. While a property is a way to allow access to a field for the external world.

And that's when auto-implemented properties enter. Because normally, it's necessary to make something like:

class Person
{
	private string _name;
	public string Name
	{
		get
		{
			return this._name
		}
		
		set
		{
			this._name = value;
		}
	}
}

while, using auto-implemented properties, one can reduce it to:

class Person
{
	public string Name { get; set; }
}

because C# compiler will automatically create the private field.

In this sense, a auto-implemented property is useful when there'll be no custom actions - the getter will only return the result and the setter will only set it, without any further validation. But when some sort of alternative action will be taken, one should implement their own version of get and set as demonstrated on example #1.
