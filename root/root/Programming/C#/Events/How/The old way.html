The old way

Firstly, one needs to define a delegate. The signature convention for the delegate method is: its name should be the action that will call the delegate on the correct verb tense + EventHandler. For instance:

The delegate should take an object called source and an EventArgs args object.

Nomenclature examples:

  • public delegate void MessageSentEventHandler(object source, EventArgs args);
  • public delegate void SongPlayingEventHandler(object source, EventArgs args);
  • public delegate void BeforeVideoEncodeEventHandler(object source, EventArgs args);

 

After that, one needs to create the event itself based on the delegate created before. The default naming convention is: the same name as the delegate created before without the EventHandler suffix. Note that it should return an object of type [created delegate].

Nomenclature examples:

  • public event MessageSentEventHandler MessageSent;
  • public event SongPlayingEventHandler SongPlaying;
  • public event BeforeVideoEncodeEventHandler BeforeVideoEncode;

 

Finally, one should create a method to actually call the event handler. It signature should look more or less like this: it should be a protected virtual void which name is the same as the event created before with the On suffix. That method calls the created event, sending the source and the EventArgs object. Note that the event method defined above is only a delegate. It isn't a method, only a pointer to one. Therefore, it's a best practice to check if the event delegate isn't null before calling it, avoiding an exception.

Nomenclature examples:

  • protected virtual void OnMessageSent;
  • protected virtual void OnSongPlaying;
  • protected virtual void OnBeforeVideoEncode;
