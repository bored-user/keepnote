Why?

Generics are the solution to a problem that existed: when one wanted to create, say a List of object X, he needed to create a XList. If it was, later, necessary to create a List of object Y, a YList had to be designed. One of the attempts to solve this problem was to create one single List class which encapsulated an object array - since object is the base class for all other classes. However, the boxing and unboxing that had to be made affected performance. Later, the <T> generics notation was created and is used until this day.
