struct vs class

A struct and a class may seem really similar at fist, but they have fundamental differences.

Â 

A struct follows a value type, while a class, a reference type.

This means that structs are copied when passed or assigned, while classes are passed around via pointers.

structs are copied on write and classes are automatically reference counted (when writing on a (passed) structs, a bitwise copy is made. When writing on (passed) classes, only the original one is modified, modifying all its pointers).

structs are built for functional programming, while classes are for object-oriented programming.

structs have no inheritance and classes have single inheritance.

The init function initialises all vars on structs and none on classes.

Data's mutability inside structs needs to be explicitly stated, while always allowed on classes.
