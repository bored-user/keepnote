Folder structure

Go programs follow a particular folder structure. It all depends on the $GOPATH environment variable (which can be echoed using go env GOPATH). When not set, $GOPATH points to /home/[user]/go in Linux and /Users/[user]/go in MacOS.

The basic folder structure for go programs is as follows:

$GOPATH/
	src/							// where the code lives
		example.com/					// site, like github.com
			user/					// user, like bored-user
				mod1/				// name of the module
					pkg1/			// name of the package
						tool1.go
						tool2.go
						tool3.go
					pkg2/
					go.mod			// info on module
					
				mod2/
					pkg1/
						tool1.go
						tool2.go
					pkg2/
	bin/							// binaries
	pkg/							// external pkgs

Note, however, that it's possible to build and compile projects outside of the illustrated folder structure. For instance, one may have a script.go file on the Desktop and run it using go run script.go or build it using go build (which will generate a binary file with the parent folder name).
