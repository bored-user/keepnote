char arrays

char arrays are special because of its properties. First, it is the only type of array that can be directly printed to the terminal using std::cout (displayed as a string).

Before explaining further, I must define C string: it is an array of characters ending in a NULL terminator (\0). 

Furthermore, there are some nuances that must be considered:

  • char message[2] { ‘h’, ‘i' }; - not considered a C string
  • char message[3] { ‘h’, ‘i’ }; -  is a C string (notice that it was initialized with 3 elements, however only two were given. Because of that, the compiler will add a NULL terminator at the end)
  • char message[] { ‘h’, ‘i’ }; - not a C string (the array is initialized automatically with the numbers of elements it was given. Because of that, this case gets converted into the first case)
  • char message[3] { ‘h’, ‘i’, ‘\0’ }; - is a C string because it ends explicitly with a NULL terminator
  • char message[] { “hi” }; - is a C string because the compiler will add the NULL terminator automatically; notice the "hi", instead of 'hi'.
