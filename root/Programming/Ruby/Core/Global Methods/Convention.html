Convention

According to Ruby convention rules, every method that has a question mark in it's name (e.g include?) returns a boolean.

 

Also, almost every ‘action method’ (e.g sort, reverse, upcase, etc) has a modifying (aka dangerous) and a non-modifying (aka non-dangerous) version. The modifying version modifies the data structure it was called on and has a ! on its name (e.g sort!, reverse! and upcase!). While the non-modifying version doesn't has an exclamation mark on it and only returns a copy, a modified version of the data structure it was called on. However there are some methods (like push, pop, shift and unshift) which do modify said data structure but don't include the bang in its name.
