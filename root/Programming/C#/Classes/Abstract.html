Abstract

Abstract classes are used to define something that's a base to other things. For instance, in a mathematical software, a shape could be defined as an abstract class - because there are shapes called triangles and circles but there's no shape called shape.

In this sense,

  • abstract classes cannot be instantiated
  • if a method is declared as abstract, the whole class needs to be abstract as well
  • a abstract class can contain, for instance, static methods/ properties
  • a abstract class may contain not abstract methods - if it's the same for all implementations
  • all abstract methods declared on an abstract class must also be defined on the derived class
  • a abstract class a a way to force other developers to adopt your style, so to speak
  • methods from a derived abstract class must use override to provide the methods' body
