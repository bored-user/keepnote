using Directive

The using keyword as a directive is used to shorten the hierarchy of data in a C# program. Essentially, using X means that the compiler should look into the X data structure when trying to find a directive. It's the default using System; directive.

To illustrate:

using System;
using System.Collections.Generic;

class Program
{
	static void Main()
	{
		Dictionary<string, string> dict = new Dictionary<string, string>();
		dict["entry1"] = "value1";
		Console.WriteLine(dict["entry1"]);
	}
}

is the same as

class Program
{
	static void Main()
	{
		System.Collections.Generic.Dictionary<string, string> dict = new System.Collections.Generic.Dictionary<string, string>();
		dict["entry1"] = "value1";
		System.Console.WriteLine(dict["entry1"]);
	}
}

but the first one is far more concise than the last.

Note that it's not possible to use using directives as shortcuts to other using directives. For instance, one cannot make using System; and using Collections.Generic; - because using directives must contain the full path to the assembly reference.

Finally, be aware that it's only possible to reference namespaces. In order to reference classes, one must use the using static directive.
