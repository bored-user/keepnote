Interfaces

Interfaces are like contracts which classes must follow. They are, in a sense, very similar to abstract classes - but have some key differences:

  • a class can only inherit from one class but can implement multiple interfaces
  • interface names start with I and the name of the interface in camel case (e.g IMyInterface)
  • a abstract class is, by name, a class - while an interface is only a contract
  • all methods in interfaces must be declared in classes that implement it

The only bad side of interfaces is that they require a lot of work to be changed, since every method on an interface must be declared on the class that's implementing it - while abstract classes do not. Generally, there's a tendency to favour abstract classes over interfaces.

Finally, a good way to know whether one needs an interface or an abstract class is to know if there's a is-a relationship between the classes (in which case one should use an abstract class) or a can-do relationship (in which case interfaces are preferred).
