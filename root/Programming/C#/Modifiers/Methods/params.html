params

The params keyword act much like the rest operator in JavaScript. It's possible to give a method (which is expecting an array of <T> as argument) lots of <T> data (not in an array) and it'll join them together, forming the array.

For instance - note the object initialization operator -, this

static void Main()
{
	Console.WriteLine(Sum(new int[] { 1, 2, 3, 4, 5 }));
}

static int Sum(int[] nums)
{
	int res = 0;
	foreach(int n in nums)
		res += n;
		
	return res;
}

could be converted to this

static void Main()
{
	Console.WriteLine(Sum(1, 2, 3, 4, 5));
}

static int Sum(params int[] nums)
{
	int res = 0;
	foreach(int n in nums)
		res += n;
		
	return res;
}

The second version is better since it's not necessary to create a new array in memory for every method call.

Note that, when using the params modifier, the method argument must be an array. Therefore, if the method used to receive argument DataType dt, one must adapt it to DataType[] dt.
