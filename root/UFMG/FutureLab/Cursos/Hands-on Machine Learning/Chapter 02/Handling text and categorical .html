Handling text and categorical attributes

Sometimes, data has non-numerical attributes. For instance, if there's a limited number of categories (e.g ocean proximity). 

When that happens, one can use sklearn.preprocesing.OrdinalEncoder() and .fit_transform(housing[['ocean_proximity']]). That turns each category into a number (from 1 to inf). That's bad because the model may learn that the higher the number, the better - and that's not always the case.

An alternative is sklearn.preprocessing..OneHotEncoder(sparse=True) (with the same .fit_transform()). That turns the data into a sparse matrix (ie, if spare=False, it'll turn it into a numpy dense array).
