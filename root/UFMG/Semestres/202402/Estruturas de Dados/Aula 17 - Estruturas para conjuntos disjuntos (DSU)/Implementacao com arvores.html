Implementacao com arvores

Na implementacao com arvores, cada conjunto eh uma arvore e o representante de cada conjunto eh a raiz daquela arvore. Todos os elementos tem um ponteiro “pai” e o pai da raiz eh si proprio.

Ou seja, a funcao find agora percorre a arvore a procura da raiz (elemento cujo pai eh ele mesmo), O(log n).

Para unir dois conjuntos, basta conectar as duas arvores (fazendo o ponteiro de uma das raizes apontar para a outra). No entanto, essa juncao sem criterios pode degenerar a arvore em uma lista encadeada. Portanto, para garantir a complexidade logaritimica para o find, deve-se garantir que a arvore permaneca balanceada. Entao, usa-se a altura da arvore, adicionando, em cada elemento, o campo rank, que guarda 0 inicialmente (altura daquele elemento). Para determinar a nova raiz, usa-se a raiz original de maior rank.
