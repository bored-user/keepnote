Metodo da bolha

Percorre o arquivo inteiro trocando elementos adjacentes, ate que os registros fiquem ordenados, ie primeiro loop de 0-n e segundo loop de 1-(sz-i).

Ou seja, para cada elemento do vetor, compara os elementos dois a dois, movendo o maior para a direita (ordem crescente).

void Bolha(int* vet, int sz) {
	for(int i = 0; i < sz; i++) {
		for(int j = 1; j < sz-i; j++) {
			if(v[j] < v[j-1]) swap(v[j-1], v[j]);
		}
	}
}

Eh estavel por trocar elementos apenas quando um eh estritamente menor que o outro (nao trocaria dois elementos iguais, logo nao alteraria a ordem relativa entre itens duplicados).

Em cada iteracao i do primeiro loop, ha n-(i+1) iteracoes do segundo loop; ou seja, C(n) = n^2-n/2 = O(n^2) comparacoes.

As movimentacoes dependem do caso:

  • no melhor caso, o vetor ja esta ordenado e portanto ha 0 movimentacoes; M(n) = 0 = O(1)
  • no pior caso, o vetor esta inversamente ordenado e ha 3*C(n) = O(n^2) movimentacoes (o 3 depende da implementacao da funcao swap).

Apesar de simples e estavel, o metodo da bolha nao eh adaptavel em termos de comparacoes e ha muitas trocas entre itens.

Melhoria: parar o algoritmo quando nao forem efetuadas trocas no for interior:

void Bolha(int* vet, int sz) {
	for(int i = 0; i < sz; i++) {
		int trocou = 0;
		for(int j = 1; j < sz-i; j++) {
			if(v[j] < v[j-1]) {
				swap(v[j-1], v[j]);
				trocou = 1;
			}
		}
		if(!trocou) break;
	}
}

Quando nenhuma troca eh feita no for interno, significa que o os elementos de vet[0] ate vet[n-1] estao ordenados, ie, o vetor esta ordenado.
